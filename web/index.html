Design Decisions

Why No Semantic Embeddings?

Running sentence-transformer models requires downloading ~400MB models per run. Our curated 55-term skill ontology + trust-weighted matching achieves near-equivalent quality without any pre-computation or network calls.

Why Fresher-Skewed?

The challenge explicitly asks for freshers to surface. Our dual-track system guarantees 30 fresher slots, while the scoring curve ensures 0 YoE is a valid starting point — compensated by certs, GitHub, and platform assessments.

Anti-Gaming as a Multiplier

Penalties are applied as a final multiplier, not subtracted additively. This means a keyword stuffer with great skills still ranks lower — their overall score is scaled down proportionally.

Quick Start

Run It in 3 Steps

No Docker. No cloud. No API keys. Just Python and your local machine.

1

Setup Environment

Create and activate a virtual environment, then install dependencies.

bash 
# Create virtual environment
python3 -m venv venv
source venv/bin/activate    # macOS/Linux
# venv\Scripts\activate    # Windows

# Install dependencies
pip install -r requirements.txt
Requires: Python 3.9+, pip, ~50MB disk for packages
2

Run the Ranker

Point it at the candidates file and choose your output path. Grab a coffee ☕ — it takes ~5 minutes.

bash 
python rank.py \
  --candidates ./dataset/candidates.jsonl \
  --out ./submission.csv

# Optional flags:
#   --verbose   Enable debug logging
#   --top-n 50  Output only top 50 (default: 100)
Output: submission.csv — 100 rows with candidate_id, rank, score, reasoning
3

Validate Output

Run the official challenge validator to confirm your submission is well-formed and meets all spec requirements.

bash 
python dataset/validate_submission.py submission.csv

✅ Submission is valid.
# Checks: 100 rows, non-increasing scores,
# no duplicate IDs/ranks, correct CSV format
Validator checks: Exactly 100 rows · Non-increasing scores · No duplicate IDs or ranks
